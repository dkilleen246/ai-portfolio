# Workflows

Documented no-code automation workflows built with tools like Make.com — each including a flow diagram, step-by-step breakdown, and notes on design decisions.

## Structure

Each workflow includes:
- **Problem Statement** — what manual process this replaces or improves
- **Tools Used** — platforms, APIs, and AI models in the pipeline
- **Flow Diagram** — visual map of the automation steps
- **Step-by-Step Breakdown** — what each node/step does and why
- **Design Decisions** — tradeoffs made and alternatives considered
- **Results** — time saved, reliability, limitations
